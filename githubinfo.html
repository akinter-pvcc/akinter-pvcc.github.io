All About Github

  • Q. What is a version control system?
  • A. A version control system is a sytem that tracks and managages changes to software code.
  • Q. What is Git?
  • A. Git is free and open source distributed version control system used for coordinating work among programmers.
  • Q. What is GitHub?
  • A. GitHub is an Internet hosting service for software development and control using Git.
  • Q. What is the difference between Git and GitHub?
  • A. Git is a tool used to manage multiple versions of code edits that are transfered to a repository, while GitHub is a location for uploading copies of a Git repository.
  • Q. Who started GitHub and how was it started?
  • A. GitHub.com was launched in April 2008 by Tom Preston-Werner, Chris Wanstrath, PJ Hyett, and Scott Chacon
  • Q. What company owns it now?
  • A. GitHub, Inc. is currently owned by Microsoft.
  • Q. How much does a GitHub account cost?
  • A. GitHub accounts are free, but there are GitHub subscription services.
  • Q. What is the Octocat?
  • A. Octocat is a tamed ghost who is a cross between an octopus and a cat.

Git/GitHub terms:

  • repository: contains all of a project's files and each file's revision history.
  • commit: a revision to one or moe files in a repository.
  • fork: a new repository that shares code and visibility settings with the original "upstream" repository.
  • push: a command to upload local repository contect to a remote repository.
  • pull requests: a request to begin the process of merging changes in branch code with the main repository.
  • workflows: a configured automated process that can run multiple jobs in a repository.
  • issues: allows programmers to track the development progress of their work on GitHub.
  • raw button: opens a file in raw form without any HTML formatting.
  • blame button: used to examine a file line by line to see who made the last modification.
