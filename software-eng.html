Chapter 7: Software Engineering

software engineering the branch of computer science that seeks principles to guide the development of large, complex software systems
integrated development environment (IDE) combines tools for developing software (editors, compilers, debugging tools, and so on) into a single, integrated package
Association for Computing Machinery (ACM):
 purpose of ACM uniting computing educators, researchers and professionals to inspire dialogue, share resources and address the field's challenges
 benefits of ACM membership provides a wide range of resources for professionals, students and institutions, which help in sustaining life-long learning
 student ACM membership provides access to thousands of online books, courses, and tutorials covering most in-demand skills
Institute of Electrical and Electronics Engineers (IEEE):
 purpose of IEEE provides a wide range of resources for professionals, students and institutions, which help in sustaining life-long learning
 benefits of IEEE membership reduced rates on many proprietary products and services including IEEE books and eBooks, journals and articles, conferences and proceedings, standards, society memberships, and continuing education courses
 student IEEE membership provides the opprotunity to earn scholarships and grants, compete in online programming competitions, and network with professionals and other students
Software Life Cycle: Development phase
 Requirements analysis to specify what services the proposed system will provide, to identify any conditions (time constraints, security, and so on) on those services, and to define how the outside world will interact with the system
 Design creating a plan for the construction of the proposed system
 Implementation the actual writing of programs, creation of data files, and development of databases
 Testing the process of debugging programs and confirming that the final software product was compatible with the software requirements specification
modular implementation separating a program's functions into independent pieces or building blocks, each containing all the parts needed to execute a single aspect of the functionality
Unified Modeling Language (UML) modern collection of tools that has been developed with the object-oriented paradigm in mind
use case a methodology used in system analysis to identify, clarify and organize system requirements
use case diagram depicts the proposed system as a large rectangle in which interactions (called use cases) between the system and its users are represented as ovals and users of the system (called actors) are represented as stick figures
Data relationships:
 one-to-one exemplified by the association between patients and occupied private rooms in that each patient is associated with only one room and each room is associated with only one patient
 one-to-many exemplified by the association between physicians and patients in that one physician is associated with many patients and each patient is associated with one physician
 many-to-many would occur if we included consulting physicians in the physician-patient relationship making many patients associate with many physicians
design pattern a predeveloped model for solving a recurring problem in software design
Software Quality Assurance (SQA) groups which are charged with overseeing and enforcing the quality control systems adopted by an organization
Software testing:
 glass box testing the software tester is aware of the interior structure of the software and uses this knowledge when designing the test
 black box testing td>refers to tests that do not rely on knowledge of the software's interior composition
 beta testing a preliminary version of the software is given to a segment of the intended audience with the goal of learning how the software performs in real-life situations before the final version of the product is solidified and released to the market
 alpha testing a preliminary version of the software is tested at the developer's site
Documentation:
 user documentation explain the features of the software and describe how to use them
 system documentation describes the software's internal composition so that the software can be maintained later in its life cycle
 technical documentation describes how a software system should be installed and serviced (such as adjusting operating parameters, installing updates, and reporting problems back to the software's developer)
Intellectual property law:
 software license the rights of the developer are formally expressed in legal terms
 software disclaimer protects software developers against by stating the limitations of their liability
